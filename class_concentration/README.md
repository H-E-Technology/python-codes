# class concentration
教室内の集中度をビデオと音声から測定するシステム

## how to use
google colab などの環境で git clone し、`main.ipynb` を動かす

## parameter について
[config.py](https://github.com/H-E-Technology/python-codes/blob/refactor/class_concentration/class_concentration/config.py) に記載する。

| パラメータ名                   | 型       | デフォルト値 | 詳細説明                                                                                                  |
| ------------------------ | ------- | ------ | ----------------------------------------------------------------------------------------------------- |
| `POSING_SCORE_THRESHOLD` | `float` | `0.7`  | ポーズの類似度スコアの閾値。この値を下回ると「不正確なポーズ」と見なされ、スコア表示が赤くなるなど、視覚的に注意喚起されます。どの代表的ポーズにも属していないと判断される基準にもなります。        |
| `SIGMA`                  | `int`   | `9`    | ポーズのスコアにガウス平滑化を適用する際の `σ`。この値を大きくすると、スコアの急な変動が抑えられ、よりなだらかなスコア変化になります。つまり、動きが緩やかな人物でも高スコアが出やすくなります。    |
| `THRESHOLD_DIST`         | `int`   | `80`   | 異なるフレームで同一人物かどうかを識別するための距離閾値（ピクセル単位）。この値が小さいほど人物の移動に厳密になり、大きいと離れていても同一人物とみなす可能性が高くなります。人物の追跡精度に影響します。 |
| `ID_FETCH_DURATION_SEC`  | `int`   | `5`    | 最初の数秒間、ID（人物識別）のために姿勢情報を蓄積する時間。ユーザの初期位置やID決定のための安定時間として使われます。                                         |
| `ID_FETCH_INTERVAL`      | `float` | `0.5`  | 姿勢情報を何秒ごとに取得するかの間隔。小さいほど頻繁にデータを取るため、より滑らかな追跡が可能ですが、計算量が増えます。                                          |
| `WINDOW_SIZE_SEC`        | `int`   | `5`    | スコアや特徴量を算出するための時間ウィンドウの長さ。この時間の間に収集されたデータをもとに、集中度やポーズスコアなどを評価します。                                     |
